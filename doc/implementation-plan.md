# 5五将棋 モバイルWebアプリ 実装計画書

## 1. Context

5五将棋（Minishogi）のモバイル向けWebアプリを新規作成する。
同一端末での2人対戦と、CPU対戦の両方をサポートする。
プロジェクトフォルダは `go-go-shogi/` として新規に作成する。

---

## 2. 5五将棋ルール仕様

### 2.1 盤面
- 5×5マスの将棋盤
- 座標: 筋（列）は右から1〜5、段（行）は上から一〜五

### 2.2 初期配置
```
         5    4    3    2    1
一段目  [飛] [角] [銀] [金] [玉]   ← 後手（上側）
二段目  [  ] [  ] [  ] [  ] [歩]   ← 後手の歩
三段目  [  ] [  ] [  ] [  ] [  ]
四段目  [歩] [  ] [  ] [  ] [  ]   ← 先手の歩
五段目  [玉] [金] [銀] [角] [飛]   ← 先手（下側）
```

- 先手の駒: 玉(5五), 金(4五), 銀(3五), 角(2五), 飛(1五), 歩(5四)
- 後手の駒: 玉(1一), 金(2一), 銀(3一), 角(4一), 飛(5一), 歩(1二)

### 2.3 駒の種類と動き

| 駒 | 表記 | 動き方 |
|---|---|---|
| 玉（王） | 玉/王 | 全方向に1マス（8方向） |
| 金将 | 金 | 前・斜め前・横・後ろに1マス（6方向、斜め後ろ不可） |
| 銀将 | 銀 | 前・斜め前・斜め後ろに1マス（5方向、横・後ろ不可） |
| 角行 | 角 | 斜め方向に何マスでも移動可能（途中に駒があると止まる） |
| 飛車 | 飛 | 縦横方向に何マスでも移動可能（途中に駒があると止まる） |
| 歩兵 | 歩 | 前に1マスのみ |

### 2.4 成り駒

| 元駒 | 成駒 | 成駒表記 | 動き方 |
|---|---|---|---|
| 銀 | 成銀 | 全 | 金と同じ動き |
| 角 | 馬 | 馬 | 角の動き ＋ 上下左右に1マス |
| 飛 | 龍 | 龍 | 飛の動き ＋ 斜めに1マス |
| 歩 | と金 | と | 金と同じ動き |

- **金・玉は成れない**

### 2.5 成りのルール
- 敵陣に駒が入る・出る・中で動くとき、成ることができる（任意）
- 敵陣の範囲:
  - 先手の敵陣 = 一段目のみ
  - 後手の敵陣 = 五段目のみ
- **強制成り**: 行き所のない駒は必ず成る
  - 歩が最終段に進んだ場合 → 強制的に「と金」になる

### 2.6 持ち駒ルール
- 相手の駒を取ると「持ち駒」となり、自分の手番で空きマスに打てる
- 成り駒を取った場合、元の駒（成る前の状態）に戻る
- **禁止事項:**
  - **二歩**: 同じ筋に自分の歩を2枚以上置けない
  - **打ち歩詰め**: 歩を打って相手の玉を詰ますことはできない
  - **行き所のない場所**: 歩を最終段に打てない

### 2.7 勝利条件
- 相手の玉を**詰み**にした方の勝ち（合法手がない状態）
- **千日手**: 同一局面が4回出現した場合の処理はルール設定で変更可能
  - デフォルト: **後手の勝ち**
  - 選択肢: 後手の勝ち / 引き分け

## 3. デザイン方針

### 3.1 全体テーマ
- **和風・木目調**: 盤面は木目テクスチャ（CSSグラデーションで表現）
- **明るいトーン**: ライト基調の和風カラーパレット
- 効果音・アニメーションは不要

### 3.2 盤面・駒
- 盤面: 木目調の暖かい色合い（CSS グラデーションで木目を再現）
- 駒: **五角形の駒型背景** + 漢字テキスト
  - CSSの `clip-path` で五角形を描画
  - 先手は正立、後手は180°回転
  - 成り駒は赤色で表示
- フォント: **明朝体**（`"Noto Serif JP"` 等のWebフォント使用）

### 3.3 カラーパレット（参考）
- 盤面背景: 木目調（`#d4a76a` 〜 `#c8945a` 系）
- マス目の線: 濃い茶色（`#8b6914`）
- 駒の背景: 淡い木色（`#f0d9a0`）
- 駒のテキスト: 黒（`#1a1a1a`）、成り駒は赤（`#c0392b`）
- UI背景: 和紙風の淡いベージュ（`#f5f0e1`）
- アクセント: 渋い朱色・金色

## 4. 技術スタック

| 項目 | 技術 |
|---|---|
| フレームワーク | React 19 |
| ビルドツール | Vite |
| スタイリング | vanilla CSS（軽量さ重視） |
| 言語 | JavaScript (JSX) |
| 盤面描画 | DOM + CSS Grid |
| 駒描画 | CSS `clip-path`（五角形）+ 明朝体フォント |
| CPU AI | ミニマックス法 + アルファベータ枝刈り |

## 7. データ構造設計

### 7.1 盤面データ（5×5の2次元配列）
```js
// board[row][col] - row: 0-4 (一段〜五段), col: 0-4 (5筋〜1筋)
// 各マスは null（空）or { type, owner, promoted }
{
  type: 'king' | 'gold' | 'silver' | 'bishop' | 'rook' | 'pawn',
  owner: 'sente' | 'gote',   // 先手 or 後手
  promoted: false              // 成っているか
}
```

### 7.2 ゲーム状態（useGame フック）
```js
{
  board: [5][5],              // 盤面
  currentPlayer: 'sente',     // 手番
  hands: {                    // 持ち駒
    sente: { silver: 0, bishop: 0, rook: 0, pawn: 0, gold: 0 },
    gote:  { silver: 0, bishop: 0, rook: 0, pawn: 0, gold: 0 }
  },
  selectedCell: null,          // 選択中のマス { row, col } or null
  selectedHand: null,          // 選択中の持ち駒 { type } or null
  legalMoves: [],              // 現在の合法移動先
  gameStatus: 'playing',       // 'playing' | 'sente_win' | 'gote_win' | 'draw'
  mode: null,                  // 'pvp' | 'cpu'
  playerSide: 'sente',        // CPU対戦時のプレイヤー側（'sente' | 'gote'）
  positionHistory: [],         // 局面履歴（千日手判定用）
  moveCount: 0                 // 手数
}
```

### 7.3 ルール設定
```js
{
  repetitionRule: 'gote_win'   // 'gote_win' | 'draw'
}
```

## 8. 実装ステップ（詳細）

### Step 1: プロジェクト初期化 ✅ 完了
1. `go-go-shogi/` フォルダ作成
2. `npm create vite@latest . -- --template react` でプロジェクト生成
3. 不要ファイル削除、基本構成整備
4. `npm install` で依存関係インストール

### Step 2: 定数・駒定義（`constants.js`）
1. 駒の種類の定数定義
2. 駒の表示名マッピング（日本語漢字）
3. 駒の価値定義（AI評価用）
4. 初期盤面データの定義
5. 成り駒マッピング

### Step 3: 駒の移動ロジック（`pieces.js`）
1. 各駒種ごとの移動ベクトル定義
   - 短距離駒（玉、金、銀、歩）: 1マスの移動方向リスト
   - 長距離駒（角、飛）: 移動方向 + 範囲なし（盤端or他駒まで）
   - 成り駒の移動方向も定義
2. 指定マスの駒の移動候補マス計算関数
3. 盤外チェック、味方駒チェック

### Step 4: ゲームロジック（`gameLogic.js`）
1. **王手判定**: 指定プレイヤーの玉が攻撃されているか
2. **合法手生成**:
   - 盤上の駒の移動（移動後に自玉が王手にならないもの）
   - 持ち駒の打ち込み
3. **詰み判定**: 合法手が0の場合
4. **二歩チェック**: 打ち先の筋に同じプレイヤーの歩がないか
5. **打ち歩詰めチェック**: 歩を打った結果、相手が詰みになるか
6. **成り判定**:
   - 成れるかどうか（敵陣への出入り）
   - 強制成り判定（行き所のない駒）
7. **千日手判定**: 盤面のハッシュを記録し、4回同一局面で設定に応じた処理
8. **局面ハッシュ生成**: 盤面+持ち駒+手番をキーにした文字列

### Step 5: UI実装
1. **ModeSelect**: タイトル画面（モード選択ボタン4つ）
2. **CpuSetup**: CPU対戦設定画面（先手/後手選択）
3. **RuleSettings**: ルール設定画面（千日手ルール切替）
4. **Board**: CSS Gridで5×5盤面描画
   - 木目調の盤面（CSSグラデーション）
   - マス目の線描画
   - 筋と段の番号表示
5. **Cell**: 各マスのタップイベント処理
   - 選択状態のハイライト
   - 移動可能マスのハイライト（半透明の丸印）
6. **Piece**: 駒の表示
   - CSS `clip-path` で五角形の駒型背景
   - 明朝体フォントで漢字表示
   - 先手は正立、後手は180°回転
   - 成り駒は赤色で表示
7. **Hand**: 持ち駒エリア
   - タップで持ち駒を選択→空きマスをタップで打ち込み
   - 駒の種類と枚数を表示
8. **GameInfo**: 手番表示バー
9. **PromotionDialog**: 「成りますか？」ダイアログ（はい/いいえ）
10. **GameOverDialog**: 勝敗表示 + 「もう一度」「タイトルに戻る」ボタン
11. **MenuModal**: メニューモーダル（再対局・タイトルに戻る・閉じる）
12. **RulesModal**: ルール説明画面

### Step 6: ゲームフロー実装（`useGame.js`）
1. ゲーム状態の初期化
2. マスタップ → 駒選択 → 移動先タップ → 移動実行
3. 持ち駒タップ → 打ち先タップ → 打ち込み実行
4. 移動/打ち込み後の処理:
   - 成り判定 → PromotionDialog表示
   - 王手チェック
   - 詰みチェック → GameOverDialog表示
   - 千日手チェック（設定に応じた処理）
   - 手番交代
5. CPU対戦モード時: CPUの手番でCPU思考を実行（先手/後手どちらでも対応）

### Step 7: CPU対戦AI（`cpu.js`）
1. **ミニマックス法 + アルファベータ枝刈り**
2. **評価関数**:
   - 駒の価値（玉=∞, 飛=10, 角=8, 金=6, 銀=5, 歩=1）
   - 成り駒ボーナス
   - 持ち駒の価値
   - 位置評価（中央寄りの駒は高評価）
3. **探索深さ**: 4手（必要に応じて調整）
4. **CPU思考中はUI操作を無効化** + ローディング表示
5. `setTimeout` でUIブロックを防止

### Step 8: モバイル最適化
1. `<meta name="viewport">` 設定
2. タッチ操作の最適化（タップ判定エリアの十分な大きさ）
3. 盤面サイズをビューポート幅に合わせてレスポンシブに
4. ダブルタップズーム防止
5. 横向き/縦向きの両対応

## 9. 操作フロー

### 盤上の駒を動かす場合
1. 自分の駒をタップ → 駒が選択される（ハイライト）
2. 合法移動先がハイライト表示される
3. 移動先をタップ → 駒が移動
4. 成れる場合 → 成り確認ダイアログ表示
5. 手番交代

### 持ち駒を打つ場合
1. 持ち駒エリアの駒をタップ → 駒が選択される
2. 打てるマス（空きマス）がハイライト表示される
3. マスをタップ → 駒が打たれる
4. 手番交代

### 選択解除
- 別の自駒をタップ → 選択切り替え
- 選択中の駒を再タップ → 選択解除
- 移動不可能なマスをタップ → 選択解除

## 10. 検証方法

1. **初期配置確認**: 盤面が仕様通りの配置か目視確認
2. **駒の移動テスト**: 各駒種の移動が正しいか（角・飛の射線ブロック含む）
3. **成りテスト**: 敵陣への進入で成り確認が出るか、強制成りが動作するか
4. **持ち駒テスト**: 取った駒が持ち駒に入り、打てるか
5. **二歩テスト**: 同じ筋に歩が2枚打てないか
6. **打ち歩詰めテスト**: 歩を打って詰みになる手が禁止されるか
7. **王手回避テスト**: 王手がかかっている時、回避手のみが合法か
8. **詰みテスト**: 合法手がない時にゲーム終了するか
9. **千日手テスト**: 同一局面4回で設定に応じた結果になるか
10. **CPU対戦テスト**: CPUが合法手を指し、思考が完了するか（先手/後手両方）
11. **モバイル操作テスト**: スマホブラウザでタップ操作が正常か
